# Contributing

Contributions are welcome, but the project should remain methodologically disciplined.

## Good contributions
- clearer worked examples
- benchmark additions
- better failure-case analysis
- tighter plain-language rewrites
- stronger cross-disciplinary comparison files
- carefully justified philosophical additions tied to runtime value

## Contributions that need extra scrutiny
- additions that make the package more slogan-heavy
- additions that reduce everything to class language
- files that repeat textbook content without improving analysis
- speculative academic claims presented as settled doctrine

## Pull request expectation
Please explain:
1. what problem the change fixes
2. whether the fix improves runtime reasoning
3. how it affects benchmark behavior
4. whether it increases jargon or dogmatism
