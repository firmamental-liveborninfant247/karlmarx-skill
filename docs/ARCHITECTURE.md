# Architecture

## Layered design

The repository is built in five layers.

### 1. Concept layer
Files under `skills/marxist-methodology/references/concepts/` define the major Marxist-derived concepts used by the system.

### 2. Foundational layer
Files under `references/foundations/` define the basic philosophical controls that prevent shallow or static reasoning.

### 3. Method layer
Files under `references/methods/` turn the conceptual and foundational layers into runtime procedures.

### 4. Comparison and source-discipline layer
`comparisons/` and `sources/` keep the package from acting like a closed doctrinal shell.

### 5. Validation layer
`references/validation/`, `benchmarks/`, and `examples/` make the project testable.

## Runtime flow
A strong-use answer typically moves like this:
1. route the problem
2. select foundational moves
3. select engines
4. compare rival frameworks
5. produce structural diagnosis
6. identify leverage and certainty
7. validate against benchmark expectations
