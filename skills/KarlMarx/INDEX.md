# Marxist Methodology Skill: Complete Index

## Orientation

This package is designed to make AI better at **general structural analysis**.

It is **not** optimized to sound like a Marxism lecturer.
It is optimized to:
- diagnose recurring system problems,
- denaturalize what looks inevitable,
- expose hidden incentives and dependency,
- compare rival explanations,
- and end with concrete leverage points.

The package uses Marxist-derived concepts because they are strong tools for these tasks.
It does **not** assume that Marxist language is always the best language.

## Design Principles

1. **General-problem first**
   The package exists to analyze ordinary strategic and systemic problems.

2. **Plain-language default**
   It translates theory into ordinary analytical language unless theory terms improve accuracy.

3. **Comparative discipline**
   It must compare its own explanation with economics, psychology, law, management, and engineering when relevant.

4. **Anti-dogmatic guardrails**
   It includes explicit non-applicability rules and misuse warnings.

5. **Verification over rhetoric**
   It ends in interventions, indicators, and revision criteria.

6. **Scholarly modesty**
   It distinguishes settled claims from contested interpretations.

## What Was Missing Before v4.1

Earlier versions progressively fixed accuracy, operating discipline, and foundational explicitness.
v4.0 already added a real foundational layer.

What still remained under-specified were several **high-efficiency philosophical weapons** that matter directly at runtime:
- content and form
- possibility and actuality
- freedom and necessity
- abstraction and concretion
- mediation and levels
- and a stronger anti-template method: concrete analysis of concrete conditions

---

## What Is New in v4.1

### 1. Philosophical completeness is stronger
The package now covers not only the classic anti-static categories, but also several high-leverage methodological categories often missing from prompt-style Marxist frameworks.

### 2. The package is better at resisting five common failures
- trusting labels instead of operative relations
- confusing imagined futures with actualizable paths
- swinging between fatalism and voluntarism
- staying too abstract to guide a case
- jumping from macro structure to local symptom without mediation

### 3. Case-specific analysis is now more explicit
A new method file on **concrete analysis of concrete conditions** makes the package less template-driven.

### 4. The package is now closer to philosophical-systematic completeness without losing runtime usability
The new additions are intended as **method controls**, not as doctrine ornaments.

## Quality Target

The goal of this package is not maximal ideological intensity.

The goal is a high-precision, high-transferability analytical framework that can:
- stay intellectually serious,
- remain usable in normal conversations,
- avoid obvious academic errors,
- and produce answers that are structurally deeper than generic advice.
